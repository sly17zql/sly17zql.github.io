每一个查询的最大美丽值

给你一个二维整数数组 items ，其中 items[i] = [price[i], beauty[i]] 分别表示每一个物品的 价格 和 美丽值 。

同时给你一个下标从 0 开始的整数数组 queries 。对于每个查询 queries[j] ，你想求出价格小于等于 queries[j] 的物品中，最大的美丽值 是多少。如果不存在符合条件的物品，那么查询的结果为 0 。

请你返回一个长度与 queries 相同的数组 answer，其中 answer[j]是第 j 个查询的答案。

解答

  • 需要为item进行排序，
  • 维护一个小于等于price的beauty的map，因为要找到小于等于当前价格的最大美丽值，假设：如果存在price为1的beauty为10，那么，price为2的beauty为5，但是因为1可以取到10，所以price也能取到10。
  • 随后遍历query，如果query本身存在于price_to_beauty的map中，那么可以直接取值；如果不存在，便通过二分法找到最后一个小于等于target的值，回去可以最大的beauty的值。
